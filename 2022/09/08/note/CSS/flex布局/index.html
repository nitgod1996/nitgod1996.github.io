# 1.2 flex 布局（弹性盒子）

flex 意味弹性布局，用来给盒子模型提供最大的灵活性。

设置 flex 布局后，子元素的 float 、 clear 、 vertical-align 属性将失效

# 1.2.1 flex 简介

采用 flex 布局的元素，称为 flex 容器，简称 “容器”。他的所有子元素自动成为容器成员，成为 flex 项目，简称 “项目”。<font color=red > 需要在容器（父元素）通过 display 属性设置 flex 布局，然后在项目（子元素）设置 flex 属性。webkit 内核浏览器要用 -webkit-flex </font>

之前做的 echarts 项目的布局就是采用 flex 布局

容器默认存在 2 根轴：水平主轴和垂直交叉轴。主轴的开始位置（与边框的交叉点）叫 main start ，结束位置叫 main end ；交叉轴开始位置叫 cross start ，结束位置叫 cross end 。

项目默认沿主轴排列。单个项目占据的主轴空间叫 main size ，占据的交叉轴空间叫 cross size

# 1.2.2 容器（父元素）属性

需要先在容器（父元素）通过 display 属性设置 flex 布局，然后在项目（子元素）设置 flex 属性。webkit 内核浏览器要用 -webkit-flex

主要以下 6 个属性：定义在容器的 style 上

  • flex-direction ：决定主轴的方向（即项目排列方向）

    • row ：默认。水平向右

    • 其他的就不一一介绍了

  • flex-wrap ：当一条轴线排不下时的换行方式

    • nowrap ：默认。不换行

    • wrap ：换行，第一行在上方

    • wrap-reverse ：换行，第一行在下方

  • flex-flow ： flex-direction 属性和 flex-wrap 属性的简写形式，默认值为 row nowrap 。

  • justify-content ：沿轴线方向的对齐方式

    主轴：水平对齐方式

    交叉轴：垂直对齐方式

    • flex-start （默认值）：按轴线头对齐。即左对齐（主轴）或上对齐（交叉轴）
    • flex-end ：
    • center ： 居中
    • space-between ：两端对齐，项目之间的间隔都相等。
    • space-around ：每个项目两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍。

  • align-items ：垂直轴线方向的对齐方式

    主轴：垂直对齐方式

    交叉轴：水平对齐方式

    • stretch （默认值）：如果项目未设置高度或设为 auto，将占满整个容器的高度。
    • flex-start ：上对齐（主轴），左对齐（交叉轴）
    • flex-end ：
    • center ：居中
    • baseline : 项目的第一行文字的基线对齐。

  • align-content ：多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用。

    • flex-start ：与交叉轴的起点对齐。
    • flex-end ：与交叉轴的终点对齐。
    • center ：与交叉轴的中点对齐。
    • space-between ：与交叉轴两端对齐，轴线之间的间隔平均分布。
    • space-around ：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。
    • stretch （默认值）：轴线占满整个交叉轴。

# 1.2.3 项目（子元素）属性

写在子项目（被排列）的元素属性

也是主要 6 个属性：

  • order ：项目排列顺序。数值越小，排列越前

  • flex-grow ：项目放大比例（按轴的方向），默认为 0。

    echarts 项目上的 flex:5 就是这个属性

  • flex-shrink ：项目缩小比例，默认为 1

  • flex-basis ：分配多余空间之前，项目占据的主轴空间。类似 width（主轴）和 height（交叉轴）

  • flex ： flex-grow , flex-shrink 和 flex-basis 的简写，默认值为 0 1 auto 。

  • align-self ：允许单个项目有不一样的对齐方式，可覆盖 align-items 属性。
