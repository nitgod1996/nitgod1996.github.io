# 解析函数各部分

一个 function 函数主要有三部分组成：函数名、参数、函数体（代码块）

# 函数名

  • 函数名就是指向函数的指针（也可以理解为对该函数的引用地址），可以赋值给别的标识符。

  • ES6 中所有函数对象都有一个 name 属性，保存函数名

  • ECMAScript 没有重载，因为参数直接由数组表示，定义时无所谓参数数量和类型

# 参数

ECMAScript 的参数内部表现为一个数组。 ECMAScript 既不关心传入的参数个数，也不关心这些参数的数据类型。定义时如果定义了 2 个参数，但调用时只传 1 个也没事。

# arguments

arguments 是函数内属性（只能在函数内部使用），是个类数组对象。返回接受的参数的数组

这样一来，开发者想传多少参数就传多少，相当于弥补了 JavaScript 函数不能重载的缺失

箭头函数没有 arguments 属性

不过如果是被包裹在函数内部，可以调用外部函数的 arguments 。（好像是句废话）

# 修改参数

关于修改参数会不会改变原变量。

JavaScript 的函数传参都是对传参进行一个浅拷贝，

因此：

  • 基础数据类型的变量均不会发生改变
  • 引用类型的变量可能会修改内容，但引用地址（变量名指向的对象）不会变。

# 箭头函数

和 Java 的 lambda 表达式一样，主要注意以下几点

  • 有且仅有一个参数，可以不写括号

  • 只有一行代码可以不写大括号（会返回值）

    • 不能用作构造函数
    • 没有 prototype 属性

# 闭包

闭包：引用了另一个函数作用域中变量的函数，通常在嵌套函数中实现

闭包会把其包含函数的活动对象添加到自己的作用域链，因此比其他函数更占内存。过度使用闭包可能导致内存过度占用，不建议经常使用闭包。这里详细解释一下：

闭包会引用外部函数的变量，在外部函数返回匿名函数（闭包）时，内部函数（闭包）会将外部函数的局部活动对象添加到自己的作用域链中，即外部函数被另一个作用域引用了，就无法被回收。只有外部引用 temp 解除对函数的引用，才能释放内存。

闭包是面试常考的问题，这里仅做简单介绍，详细的需要自行去搜索

# Function 属性与方法（函数）

# 属性：arguments、arguments.callee、this、new.target、length、name、prototype

内部属性：只能在函数体内使用的

  • arguments ：该函数接受的参数的数组

    • arguments.callee ：返回该函数名（指向该函数的指针 / 地址）。一般用来递归

  • this ：指向某个对象作用域的指针。

  • new.target 属性：

    • 如果函数正常调用（没有用 new 实例化）： new.target 返回 undefined
    • 如果 new 实例化了对象： new.target 返回该（对象的）构造函数

对象属性：对象属性，可以用函数名调用

  • length ：返回函数定义时参数的个数

  • name ：返回函数名

  • prototype ：原型，用在面向对象上的，定义在原型上的属性和方法可以继承。

# 方法：apply ()、call ()、bind ()

  • apply() ：在指定的环境下调用函数（以指定的 this 和 参数 ）

    • this ：函数运行的指定 this ，指定以某个对象的作用域
    • arguments ：参数

    <font color=red> 注意一下，call 和 apply 输入参数不太一样，apply 需要 [] 包裹，call 不需要 </font>

    apply 和 call 最重要的是控制函数调用指定作用域 this 的能力

  • bind() ：以指定的对象域创建一个新的函数实例（也可以理解为 返回一个绑定了指定对象域的函数 ）

    • this ：新函数的指定 this

    用了 bind() 后无论以什么对象调用， this 指向都不会改变（就像箭头函数）

# 三者区别

  • apply 和 call 是以指定的 this 执行函数
    • apply：可以使用 arguments ，参数以数组形式传入
    • call：不能用 arguments ，参数以 ... 传入
  • bind：创建一个绑定了 this 的函数实例（不执行），之后该函数的执行均是在该域上

# Function 用作构造函数（面向对象）

ES6 之前 JavaScript 没有 class ，都是通过 Function 和原型 prototype 定义对象（类）的。

  • Function 定义构造函数，主要写成员变量，每个对象实例都不相同
  • prototype 主要定义成员方法（函数），所有对象实例共有

# new.target：判断函数 or 对象

Function 的内部属性 new.target 可以用来判断这个 Function 是用作函数还是对象。

ES6 新增的 new.target 属性：

  • 如果函数正常调用（没有用 new 实例化）： new.target 返回 undefined
  • 如果 new 实例化了对象： new.target 返回该（对象的）构造函数

# this：定义属性

this 表示的是对象本身，指向当前作用域

如上是一个定义 Person 对象的代码， Person 在语义上是构造函数名，但同时也是该对象（类）名。 this.name 表示这个对象的 name 属性

也可以在构造函数中通过 this 定义成员方法（函数），但一般不建议这么做。原因前面已经说过，构造函数里的内容每个对象实例都是不同的，也就是说每次通过 new 新建一个对象实例，都会执行一次构造函数，都会创建一次构造函数里的变量和方法（函数）。但一般来说同一个对象（类）共享一个函数即可，一般不为每个对象实例单独定义一个功能一样的函数。
