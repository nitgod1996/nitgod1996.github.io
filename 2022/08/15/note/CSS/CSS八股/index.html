# 说说你对盒子模型的理解

浏览器的渲染引擎会将所有元素表示为一个个矩形的盒子，盒子主要由四部分组成。

  • content ：元素内容，显示文本或图像
  • padding ：元素内边距，这是元素内容和元素边框之间的距离
  • border ：元素边框，这个边框可以设置宽度
  • margin ：元素外边距，这是元素边框和另一个元素之间的距离。

标准盒子模型中， width 定义的就是 content-box 的尺寸，怪异盒子模型是 border-box 的尺寸，也就是 content+padding+border 的宽度。通过 box-sizing 设置选用哪种模型。

# 外边距重叠

另外 margin 左右方向是元素边框与另一个元素外边距之间的距离，上下方向的话因为会产生外边距重叠，所以是两个元素边框之间的距离。两个元素的 margin 取更大的那个。还有一种情况，元素没有 border 和 padding 的情况下，其子元素会和垂直方向的元素参与外边距重叠。

避免外边距重叠，有以下避免方法：

  • 父元素 overflow: auto （或者非 visible 的值），防止和子元素折叠。这种方式副作用最小。
  • 父元素加上 border 或者 padding ，防止和子元素折叠
  • 设置为浮动元素 float、内联块 inline、绝对定位 absolute 或固定定位 fixed 时，外边距不会在它外面折叠
  • 使用 Flexbox 布局时，弹性布局内的元素之间不会发生外边距折叠。网格布局同理
  • 当元素显示为 table-cell 时不具备外边距属性，因此它们不会折叠

# css 选择器有哪些？优先级？哪些属性可以继承？

选择器主要是 id 选择器，类（ class ）选择器，伪类（ : ）选择器，属性选择器，元素（标签）选择器，伪元素（ :: ）选择器，通用（ * ）选择器，其中

  • id 选择器最大
  • 类选择器、伪类选择器、属性选择器次之
  • 元素选择器、伪元素选择器再次之
  • 通用选择器最小

还有一些组合器，比如 +、 、> 组合器没有优先级。

有多个选择器组合的时候，对应优先级的相加，先比较优先级最大的选择器的数量，然后再比第二大的...

这些是选择器的优先级。另外，行内样式优先级大于选择器，样式后面有些 !important 的优先级最大

# 哪些可以继承

主要和文本相关的大部分可以继承，比如行高、字体大小、颜色、对其方式这些。

继承的优先级很低，只要样式被重定义了，优先级就被该选择器覆盖

# 说说 em/px/rem/vh/vw 区别？

这些除了 px 都是相对单位。px 是绝对单位，表示的是像素点的大小，这就造成了在不同屏幕上显示出来的元素大小可能会大相径庭。

em = 当前元素的 font-size，rem 是根元素也就是 body 的 font-size。vh/vw 是视口的高度和宽度。

一般来说 rem 用的比较多，em 用起来有一定隐患，需要对每个元素的 font-size 了如指掌，有时候如果用 em 定义了 font-size 可能会造成尺寸的嵌套指数级变大或变小，比如设置 font-size: 0.9em，表示字体变为父辈的 90%

# 水平垂直居中

垂直居中：

  • 元素定高宽：先将父子元素定宽高

    • 使用 absolute

    • 使用 absolute+calc

    以上 2 中缺点是需要知道子元素的宽高

    • 使用 absolute+margin

  • 行内元素：

    • lineheight 设为和 height 一样实现垂直居中
    • 使用 vertical-align 垂直居中
    • 使用 text-align 水平居中

  • table-cell：

    • display 设为 table-cell，然后使用 vertical-align 垂直居中。使用 text-align 水平居中

  • flex：使用 flex 水平垂直居中

    • justify-content：center 沿着轴的方向居中
    • align-item：center 垂直轴的方向居中

# BFC

  • BFC 就是页面中的一个隔离的独立容器，容器里的标签不会影响到外部标签
  • 计算 BFC 的高度时，浮动元素也参与计算

# 怎样触发 BFC

这里简单列举几个触发 BFC 使用的 CSS 属性

  • 根元素、即 HTML 元素
  • 浮动元素：float
  • position：absolute 或 fixed
  • overflow: hidden、visible
  • dispaly：inline-block、table、flex、grid 等

# BFC 解决了什么问题

  1. 使用 Float 脱离文档流，高度塌陷：让父元素变成 BFC

  2. Margin 边距重叠

  3. 两栏布局：float 会占用第一个元素的宽度，使用 BFC 就可以隔开
