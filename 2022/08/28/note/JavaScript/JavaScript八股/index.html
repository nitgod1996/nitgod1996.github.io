# 基本数据类型和引用类型有什么区别？

  • 基本数据类型存储在栈中
  • 引用类型的对象存储于堆中

当我们把变量赋值给一个变量时，解析器首先要确认的就是这个值是基本类型值还是引用类型值

再说得详细一点：基本数据类型其变量保存的直接就是数据的值，而引用类型的变量保存的是引用地址（也就是说，引用类型把引用地址放在了栈中），引用地址指向堆中的数据，如下图

因此，复制基础变量直接就是复制了其值；而复制引用类型可能只是复制了其引用地址，这种情况，2 个变量指向的对象还是同一个。如果要完全复制，需要深拷贝

# 解释一下什么是浅拷贝，什么是深拷贝

  • 浅拷贝：就是复制的是引用类型的地址（该地址仍指向同一内存）。比如直接复制一个数组，或者用 slice 复制一个二维数组
  • 深拷贝：完全拷贝引用类型的值（值不是同一内存的），产生相同大小的额外空间.

深拷贝的实现：

  1. JSON.parse(JSON.stringify()) ：这是一种比较简单的方法，但缺点是 undefined 不能被转换并抛出异常

  2. 递归拷贝：这种方法比较可靠

# 判断数据类型方法：typeof、instanceof、constructor、new.target 的区别

typeof： typeof 主要用于基本数据类型以及函数的判别，对引用数据类型 object 就无法细分其类别了。其以字符串形式返回结果

  • undefined ：表示值未定义
  • boolean ：布尔值
  • string ：字符串
  • number ：数值
  • object ：对象或 null
  • function ：函数和 class
  • symbol ：符号

instanceof： instanceof 主要是判断后者是否处于前者的原型链上游。

可以用来判断：

  1. 对象实例是否属于该对象（伪类）
  2. 对象（伪类）是否继承于另一个对象（伪类）

<font color=red> instanceof 不适用于简单类型 </font>

constructor： constructor ：对象原型上属性，是一个指针，指向创建该对象的构造函数。可以用来判断 2 个对象是不是同一个类

constructor 是指向对象实例的构造函数

但可以通过原型 prototype.constructor 获取构造函数（因为每个类的原型就是一个原型实例对象）

new.target：独属于函数的属性，返回对象实例的构造函数。

  • 如果函数正常调用（没有用 new 实例化）： new.target 返回 undefined
  • 如果 new 实例化了对象： new.target 返回该构造函数

因为 new.target 是函数的专有属性，和 arguments 一样，因此 <font color=red> 只能在函数体内调用 </font>

# 为什么 0.1+0.2!==0.3

浮点数的精度远不如整数。如 0.1+0.2 !== 0.3，而是 0.30000000000000004

JavaScript 无法完全表示 0.1 和 0.2，而 js 则是对二进制进行截取，此时 0.1 就不再是 0.1 , 而是 0.100000000001， 而 0.2 则是 0.2000000002，那么两者相加时则大于了 0.3

  • 十进制整数转二进制方法：除 2 取余；

  • 十进制小数转二进制方法：乘 2 除整

# JavaScript 预编译

总体来看， JavaScript 编译过程只有下面三个步骤： 1. 语法分析 2. 预编译 3. 解释执行。而 JavaScript 预编译发生在代码片段执行前的几微秒（甚至更短！），预编译分为两种，一种是函数预编译，另一种是全局预编译

# 函数预编译四部曲

  1. 创建活动对象 Activation Object （以下简写为 AO 对象）；（作用域那章有说到过，就是保存函数内所有变量以及参数的对象）
  2. 找形参和变量声明，将变量声明和形参作为 AO 的属性名，值为 underfined ；
  3. 将实参和形参值统一；
  4. 在函数体里找函数声明，将函数名作为 AO 对象的属性名，值赋予函数体。
  • 形参：函数声明时的参数，可以认为是个参数名，没有实际的值。或者说可以看成是保存实参的变量名
  • 实参：函数调用时的传参，是一个确切的值，当传参是引用类型是，实参是其地址。在函数内部，由形参（代表变量名）保存实参

也就是说，先将变量声明和形参提升，再形参赋值，然后对函数声明整体提升。而对 var 变量赋值则不在预编译阶段进行

# 全局预编译三部曲

  1. 创建全局对象 Global Object （以下简写为 GO 对象）；
  2. 找变量声明，将变量声明作为 GO 的属性名，值为 undefined ；
  3. 找函数声明，将函数名作为 GO 对象的属性名，值赋予函数体。

总的来说和函数预编译差不多，就是少了个形参

# == 和 === 的区别

  • == ：等于操作符，会做类型转换进行比较（一般是转为数值）

    • 两个都为简单类型，字符串和布尔值都会转换成数值，再比较
    • 简单类型与引用类型比较，对象转化成其原始类型的值（调用对象的 valueOf() ），再比较
    • 两个都为引用类型，则比较它们是否指向同一个对象
    • null 和 undefined 相等
    • 存在 NaN 则返回 false

    valueOf() ：返回对象，通常与 toString 返回一致。

  • === ：全等操作符。需要类型相同，值也相同

    • '55' !== 55
    • null === null ， undefined === undefined 。但 null !== undefined

# 说说对内存回收的理解

JavaScript 会自动释放内存，其基本思路是：

  • 确定哪些变量不会再被使用，然后释放他的内存
  • 周期性：垃圾回收程序是每隔一段时间运行一次的

如何标记未使用的变量主要有以下两种方式：标记清理和引用计数

# 标记清理（常用，针对变量）

标记清理是针对变量（变量名）的

  1. 当变量进入作用域时（如函数内声明一个变量），这个变量会被加上一个存在于作用域的标记。

  2. 当变量离开作用域时（作用域内的代码执行完，离开执行栈），也会被加上一个离开作用域的标记。

  3. 垃圾回收程序运行时，会标记（待删除）内存中存储的所有变量。然后将

    • 所有作用域中的变量
    • 被在作用域中的变量引用的变量

    的标记去掉，剩下的还带着（待删除）标记的变量就是待删除的了。（因为不会再被访问了）

  4. 到了回收周期进行清理待删除标记的变量，并回收他们的内存。

# 引用计数（针对值）

引用计数是针对值（变量值）的

  • 当声明一个变量并为它赋予一个引用值时，这个值的引用数为 1

    • 如果这个值又被赋给了另一个变量，那么引用数 + 1

  • 如果保存该值的变量被覆盖了，则引用数 - 1

    • 或者对该值引用的每个变量离开作用域（作用域内的代码执行完，离开执行栈），都 -1

  • 当引用数为 0 时，在下次回收时被销毁

引用计数有一个严重的缺陷，就是当 a 引用 b，而 b 又引用 a 时，会造成循环引用，导致这两个值一直不被释放

# 总结

  • 标记清理是针对变量（变量名）的
  • 引用计数是针对值（变量值）的
  • 标记清理比引用计数更常用，也更好

# 回收周期

回收频率太低会导致垃圾积压，频率太高会到处小内存片泛滥。

IE7 后，JavaScript 引擎的垃圾回收程序为动态改变分配变量、字面量或数组槽位等阈值。

  • 设一个初始阈值。

  • 如果有一次回收内存不到已分配的 15%，则阈值会翻倍。

  • 如果有一次回收内存达到 85%，则阈值重置为默认值。

（有点像网络拥塞控制的慢开始和快重传）

# 说说你对作用域及作用域链的理解

任何变量（不管包含的是原始值还是引用值）都存在于某个执行上下文中（也称作用域）。

作用域决定了变量的生命周期，以及他们可以访问代码的哪些部分。

JavaScript 有 3 种作用域：

  • 全局作用域：最外层作用域，即 window 对象的作用域
  • 函数作用域：函数内的作用域，也是最复杂的一个作用域。
    • var 的声明范围即为函数作用域
  • 块级作用域： {} 包裹的作用域，如 for 、 while 、 if 等块。单独的 {} 块也算
    • 函数作用域也算块级作用域
    • let 和 const 的声明范围是块级作用域

每个作用域都有一个关联的变量对象，保存该作用域下定义的所有变量和函数。

如果是函数作用域，其一开始就有个定义对象 arguments 作为变量对象

# 函数作用域

每个函数调用都有一个专属的函数作用域，其所有代码都执行完毕后会销毁。

当代码执行流进入函数，函数的作用域被推到一个作用域栈（上下文栈）上（此时原本的作用域比如全局作用域就被压在了下面），执行完毕后弹出，继续执行原来的作用域的代码。

    2. 在第 3 行定义了一个 changeColor 的 Function 对象
  2. 在第 7 行执行 changeColor() ，代码执行流进入函数 changeColor() 的作用域

  4. 代码执行流重新进入全局作用域，继续往下执行

# 作用域链

作用域链：说白了就是一个作用域的嵌套关系，决定了变量取值的查找顺序，类似原型链。

JavaScript 引擎在查找变量时顺着作用域链找的，先找最前端的正在执行的作用域的变量对象，没找到再一层层往下找，直到全局作用域。因此，<font color=red> 访问局部变量比访问全局变量要快得多。</font>

比如一段代码，里面有一个函数 1，函数 1 里面定义了一个函数 2。执行的时候，函数 2 里面的变量首先看一下函数 2 作用域里有没有对其赋值，没有的话看函数 1 里面有没有对这个变量赋值，再没有就看全局作用域有没有对这个变量赋值。当前作用域（函数 2）→上一层作用域（函数 1）→...→全局作用域，这就是一个作用域链，也是变量取值的一条查询链。

从结构上来看，所有的作用域链组成一个类似树的结构，全局作用域为根节点，定义来全局作用域的作用域（函数域、块域）为孩子节点（同级），各自作用域内定义的新的作用域又为其孩子节点，以此类推。每个节点的变量到根节点的链路均是其作用域链。

# 说说对原型及原型链的理解

原型链：说白了就是继承链，其体现了一个继承关系。决定了变量取值的查找顺序，类似作用域链

当要找一个对象的属性值时，

  1. 首先看这个对象作用域的属性值（比如构造函数内有没有对其赋值、 new 的时候有没有对其赋值，创建后有没有对这个属性进行赋值。这些都会影响其作用域的属性值）；

  2. 如果没有，则去其原型对象上找，看一下原型对象上有没有默认的值（原型对象是该类所有对象共享的）。（如果有父类，原型对象一般是父类的对应属性值）；

  3. 如果没有，再去其原型的原型对象（父类的原型对象 / 父类的父类）找；

  4. 还是没有，依次往上找，直到 Object

# 原型的 2 大属性：constructor 和_proto_

  • prototype.constructor ：隶属于 prototype 的属性，指向构造函数。（也可以理解为指向类）
  • prototype.__proto__ ：隶属于 prototype 的属性，指向构造函数的原型对象。即对象的原型

<font color=red> __proto__ 也可以说是指向原型链中的上一级原型。因此 __proto__ 是原型链上至关重要的属性，原型链的往上遍历就是通过 __proto__ 实现的。</font>

所以顺着原型链找就是通过 __proto__ 实现的

# 说一说继承吧

  • 原型链继承
  • 构造函数继承（借助 call）
  • 组合继承
  • 原型式继承
  • 寄生式继承
  • 寄生组合式继承

# 原型链继承

原型继承父类实例（构造函数）。

存在问题：

原型不应当为构造函数，因为构造函数一般写实例特有的参数，而原型一般写类共有的函数，这样子继承会使得子类的属性共享。

<font color=red> 这种方法可以继承父类方法，因为父类实例可以调用父类方法 </font>

# 盗用构造函数继承

子类构造函数中调用父类构造函数（类似 Python 的 super）

缺点：

只继承了参数，没继承方法

# 组合式继承

上述两种组合。原型继承父类实例，构造函数继承构造函数

缺点：要 new 2 次父类实例

# 原型式继承

对传入对象进行拷贝（一般通过 Object.create() ）

缺点：原型是对传入对象的浅拷贝，多个实例的引用类型属性指向相同的内存，存在篡改的可能

# 寄生式继承

对原型式继承的增强（感觉好像就是多加了个 get 函数啊）

缺点：和原型式继承一样，浅拷贝属性会串

# 寄生式组合继承

  • 寄生式继承：子类原型继承父类原型
  • 构造函数继承：子类构造函数继承父类构造函数

拷贝一下原型，不能直接 Child.prototype=Father.prototype, 因为 prototype 是对象，是引用值，如果子类重写方法，那么父类的方法也会变化。所以方法不能重写.
但浅拷贝一下足够，因为方法可以共用，重写方法的时候会新建堆内存，地址不和父类的一样就行
