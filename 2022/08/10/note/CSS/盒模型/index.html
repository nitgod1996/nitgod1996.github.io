# css 基础框盒模型（盒子模型）

CSS 传统布局基于盒子模型，所有元素都在一个盒子（容器）中

每个盒子有 4 个边界：

  1. 内容边界 Content edge（最里面显示内容）
  2. 内边距边界 Padding Edge（content + 内边距 padding）
  3. 边框边界 Border Edge（padding edge + 边框宽度 border width）
  4. 外边框边界 Margin Edge（border edge + 外边距 margin）

# 1. 内容边界

容纳着元素的 “真实内容”，例如文本、图像等，其尺寸为内容宽度和内容高度确定。

主要 css 属性：

  • width ：内容宽度。% 或 px 或 rem。% 是相对父元素的百分比

  • height ：内容高度。% 或 px 或 rem

  • min-width ：最小内容宽度。这种一般是针对可以灵活变动的元素

  • max-width ：最大内容宽度

  • min-height ：最小内容高度

  • max-height ：最大内容高度

  • box-sizing ：

    • content-box ：默认。在宽度和高度之外绘制元素的内边距和边框。<font color=red>height 和 width 绑定内容边界 </font>
    • border-box ：向内绘制内边距和边框。换句话说，<font color=red>height 和 width 绑定边框边界 </font>

# 2. 内边距边界

内容边界 + padding。是边框（border）内的那块空间。内边距则是内容和边框之间的那块空白区域

主要 css 属性：

  • padding-top ：上内边距。px、%、rem
  • padding-right ：右内边距。
  • padding-bottom ：下内边距
  • padding-left ：左内边距
  • padding ：内边距，以上的简写

# 3. 边框边界

内边距边界 + 边框宽度。边框由 border 属性控制

主要 css 属性：

  • border-width ：边框宽度。
  • border-style ：边框样式：
    • dotted 定义点状
    • dashed 定义虚线
    • solid 定义实线。
    • double 定义双线
  • border-color ：边框颜色
  • border ：以上简写
  • 可以分别控制各边边框
    • border-top ：上边框
      • border-top-width ：上边框宽度
  • box-sizing ：如果 box-sizing 设置 border-box ，则可用 width 和 height 设置边框边界

# 4. 外边距边界

盒子最外围边界，边框边界 + margin。外边距是边框与别的盒子之间的空白区域

主要 css 属性：

  • margin-top ：上外边距。px、%、rem
  • margin-right ：右外边距。
  • margin-bottom ：下外边距
  • margin-left ：左外边距
  • margin ：外边距，以上的简写

# 4.1 负外边距

不同于内边距和边框宽度，外边距可以设置为负值。负外边距有一些特殊用途，比如让元素重叠或者拉伸到比容器还宽。

虽然负外边距在有些时候很实用，不过还是要尽量避免，不然网页样式可能会失控。

# 4.2 外边距折叠

当顶部和底部的外边距相邻时，就会重叠，产生单个外边距。这种现象被称作折叠。折叠规则是取较大的边距。（其实就是把外边距视为自己边框 - 相邻盒子边框的距离）。<font color=red> 只有上下外边距会重叠，左右不会 </font>

如：段落（ <p> ）默认有 1em 的上外边 距和 1em 的下外边距。但当前后叠放两个段落时，它们的外边 距不会相加产生一个 2em 的间距，而会折叠，只产生 1em 的间隔。

折叠外边距就像 “个人空间”。如果在公交车站站着两个人，他们每个人都认为较为舒适的 个人空间应为 3 英尺 ①，那么他们就会乐意间隔 3 英尺，而不必间隔 6 英尺才让双方满意。

# 4.2.2 避免外边距折叠

外边距折叠从某种意义上来说也是外边距扩展。因为当父元素下面紧挨着的子元素外边距大于父元素的外边距时，父元素的外边距会变大（和子元素一样），因此有时候避免外边距折叠是很有必要的。

有以下避免方法：

  • 父元素 overflow: auto （或者非 visible 的值），防止和子元素折叠。这种方式副作用最小。
  • 父元素加上边框或者内边距，防止和子元素折叠
  • 设置为浮动元素 float、内联块 inline、绝对定位 absolute 或固定定位 fixed 时，外边距不会在它外面折叠
  • 使用 Flexbox 布局时，弹性布局内的元素之间不会发生外边距折叠。网格布局同理
  • 当元素显示为 table-cell 时不具备外边距属性，因此它们不会折叠

# 容器尺寸问题

容器的宽度默认由其性质决定（块级元素还是行内元素）；容器的高度默认由内容决定， 而不是容器自己决定。

# 1. 溢出问题

当明确设置一个容器的尺寸时，内容可能会溢出容器。如下

这种情况可以通过 overflow 属性控制

  • visible （默认值）：溢出可见。 所有内容可见，即使溢出容器边缘
  • hidden ：溢出隐藏。溢出容器内边距边缘的内容被裁剪，无法看见。
  • scroll ：溢出滚动。容器出现滚动条，用户可以通过滚动查看剩余内容。
  • auto ：只有内容溢出时容器才会出现滚动条。

可以用 overflow-x 属性单独控制水平方向的溢出，或者用 overflow-y 控制垂直方向 溢出。

# 2. 百分比宽高

百分比参考的是元素容器块（其实就是父元素）的大小，而容器的高度通常是由 子元素的高度决定的（如果没有显示的设置过的话）。那么如果使用百分比设置子元素的高度，那么就会造成死循环，就会被浏览器忽略

因此，要想让百分比高度生效，必须显示地声明容器（父元素）的高度

# 等间距排列（猫头鹰选择器）

经常会有这样的场景：一排组件（如卡片）需要等间距的排列，最左边和最右边都紧贴屏幕或外部容器（没有最左外边距和最右的外边距）。

这就有了个问题，统一设置外边距总有一边是会有空白的，有时候会把所有都设置一个外边距，然后到最边上的一个元素用 0 覆盖。其实有个更加方便的方法就是使用猫头鹰选择器 * + * 。

* + * ：选中所有元素的除了兄弟中排第一的所有元素。

这样一来，就可以设置除了第一个之外的所有卡片的左边距即可。

当然，也可以只用 + * ，针对性地选中对应容器下的元素
