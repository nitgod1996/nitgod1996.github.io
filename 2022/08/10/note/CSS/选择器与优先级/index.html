# 选择器

# 基础选择器

  • #id -id 选择器：通过 id 属性和 #+id值 匹配，优先级为 1,0,0
  • .class - 类选择器：通过 class 属性和 .+class值 匹配，优先级为 0,1,0
  • tagname - 标签（元素）选择器：通过 标签名 匹配（直接写标签名即可），优先级 0,0,1
  • * - 通用选择器：通过 * 匹配，匹配所有元素，优先级 0,0,0

# 属性选择器

属性选择器用于根据 HTML 属性匹配元素。其优先级与一个类选择器（0,1,0）相等

  • [attr] ： 匹配拥有指定属性的元素，无论属性值是什么，例如： input[disabled] 。
  • [attr="value"] ：匹配特定属性且特定属性值的元素， 例如： input[type="radio"] 。
  • [attr^="value"] -“开头” 属性选择器：匹配特定属性以特定字符串开头的属性值的元素，例如： a[href^="https"] 。
  • [attr$="value"] -“结尾” 属性选择器。同上
  • [attr*="value"] -“包含” 属性选择器。 且属性值包含指定的字符串值
  • 后面的也主要是一些字符串匹配规则
  • [attr~="value"] ——“空格分隔的列表” 属性选择器。该选择器匹配的元素拥有指定 属性 attr，且属性值是一个空格分隔的值列表，列表中的某个值等于指定的字符串值， 例如：a [rel="author"]。
  • [attr|="value"] —— 匹配的元素拥有指定属性 attr，且属性值要么等于指定的字符 串值，要么以该字符串开头且紧跟着一个连字符（-）。适用于语言属性，因为该属性有 时候会指定一种语言的子集（比如墨西哥西班牙语，es-MX，或者普通的西班牙语，es）， 例如：[lang|="es"]

属性选择器不区分大小写

# 组合器

组合器将多个基础选择器连接起来组成一个复杂选择器，组合器本身不增加优先级

  • （空格）- 后代组合器：匹配该元素的所有子孙元素
  • > - 子组合器：匹配该元素的直接后代
  • + - 相邻兄弟组合器：匹配该元素紧随其后的一个兄弟元素
  • ~ - 通用兄弟组合器：匹配该元素后面所有的兄弟元素

多个组合器可以复合使用

# 伪类选择器 :

伪类选择器和伪元素选择器是比较难以理解的部分：

  • 伪类选择器：处于某个特定状态的元素。如用户交互，或者相对于父子兄弟的位置
  • 伪元素选择器：匹配在文档中没有直接对应 HTML 元素的特定部分。如某个文本前面的位置

伪类选择器种类：

用户交互

  • :focus ：匹配获取焦点的元素
  • :hover ：匹配鼠标正悬停在其上方的元素

文档树

  • :root ：匹配文档根元素

  • :first-child ：匹配该元素所有子孙中的老大

  • :last-child ：匹配该元素所有子孙中的老小

  • :only-child ：匹配该元素所有子孙中的独生子女。

  • :nth-child(an+b) ：匹配该元素所有子孙中一定位置的子孙。

:first-of-type—— 类似于:first-child，但不是根据在全部子元素中的位置查找元 素，而是根据拥有相同标签名的子元素中的数字顺序查找第一个元素。  :last-of-type—— 匹配每种类型的最后一个子元素。  :only-of-type—— 该选择器匹配的元素是满足该类型的唯一一个子元素。  :nth-of-type (an+b)—— 根据目标元素在特定类型下的数字顺序以及特定公式选择元 素，类似于:nth-child。  nth-last-of-type (an+b)—— 根据元素类型以及特定公式选择元素，从其中最后一个 元素往前算，类似于:nth-last-child。  :not ()—— 匹配的元素不匹配括号内的选择器。括号内的选择器必须是基 础选择器，它只能指定元素本身，无法用于排除祖先元素，同时不允许包含另一个排除 选择器。  :empty—— 匹配的元素必须没有子元素。注意，如果元素包含空格就无法由该选择器 匹配，因为空格在 DOM 中属于文本节点。写作本书时，W3C 正在考虑:blank 伪类选择 器，它跟:empty 的行为类似，但是能选中仅包含空格的元素，目前还没有浏览器支 持:blank。

:disabled—— 匹配已禁用的元素，包括 input、select 以及 button 元素。  :enabled—— 匹配已启用的元素，即那些能够被激活或者接受焦点的元素。  :checked—— 匹配已经针对选定的复选框、单选按钮或选择框选项。  :invalid—— 根据输入类型中的定义，匹配有非法输入值的元素。例如，当的值不是一个合法的邮箱地址时，该元素会被匹配（Level4）。  :valid—— 匹配有合法值的元素（Level4）。  :required—— 匹配设置了 required 属性的元素（Level4）。  :optional—— 匹配没有设置 required 属性的元素（Level4）。以上并未列出全部伪类 选择器。请参阅 MDN 文档 Pseudo-classes，查看 MDN 上的完整清单。

# 伪元素选择器 ::

伪元素类似于伪类，但是它不匹配特定状态的元素，而是匹配在文档中没有直接对应 HTML 元素的特定部分。伪元素选择器可能只匹配元素的一部分，甚至向 HTML 标记中未定义的地方 插入内容。

语法：前面是一个正常的选择器，后面跟上一个伪元素选择器

  • ::before ：在该元素前创建一个伪元素。该元素默认是行内元素，可用于插入文字、图片或其他形状。必须指定 content 属性才能让元素出现，例 如： .menu::before 。
  • ::after ：在该元素后创建一个（伪）元素。
  • ::first-letter ：匹配该元素的第一个文本字符的样式，例如： h2::firstletter 。
  • ::first-line ：匹配该元素的第一行文本的样式。
  • ::selection ：匹配指定用户使用鼠标高亮选择的任意文本的样式。通常用于改变选 中文本的 background-color。只有少数属性可以使用，包括 color、background-color、 cursor、text-decoration

# 样式冲突（层叠）

每条规则单独来看很简单，但是当两条规则提 供了冲突的样式时会发生什么呢？如果你发现有一条规则没有按照预期生效，可能是因为另一条 规则跟它冲突了。

比如某个元素（标签），在不同的地方都设置了样式（比如不同的选择器中），这时候可能会造成样式冲突。比如都定义了一样的样式如字体颜色，或者复杂点的通过布局等形成冲突。那么我们得知道到底哪个样式生效

如上例子，我们在 3 个不同的选择器中定义了例子中的元素的字体，同时还在标签内通过 style 属性也定义了，那么到底哪个生效呢？

层叠指的就是这一系列规则。它决定了如何解决冲突，是 CSS 语言的基础。当声明冲突时，层叠会依据三种条件解决冲突。

  1. 样式的来源：样式是从哪里来的，包括你的样式和浏览器默认样式等。

  2. 选择器优先级：哪些选择器比另一些选择器更重要。

  3. 源码顺序：样式在样式表里的声明顺序。

# 样式的来源

即 样式来自哪里？你添加到网页里的样式并不是浏览器唯一 ** 使用的样式，还有其他类型或来源的样式：

  • 浏览器默认样式（用户代理样式）
  • 作者样式（你定义的样式）
    • 行内样式（内联样式）：即通过 style 属性直接定义在标签内的样式
    • 选择器

优先级：作者样式表 > 浏览器默认样式。行内样式 > 选择器

新建的 HTML 文档一开始在各种浏览器中都会有其默认的样式（如默认字体、边距等），而程序员定义的样式会覆盖浏览器默认的样式

# 优先级

遇到冲突时，浏览器会尝试检查它们的优先级，优先级高的享受渲染权

前面已经说过，优先级：作者样式表 > 浏览器默认样式。行内样式 > 选择器

但有个例外，就是 !important 声明

# 选择器优先级

选择器内部也有优先级的划分，一般来说，id 选择器的优先级最大

选择器优先级有 4 个等级：从高到低如下

  1. id 选择器（ # ）:1,0,0
  2. class（类）选择器（ . ）、属性选择器（ [] ）、伪类选择器（ : ）：0,1,0
  3. 标签（元素）选择器、伪元素选择器（ :: ）：0,0,1
  4. 通用选择器（ * ）：0,0,0

同一级别的选择器优先级可以相加：因为有组合器的存在，因此有时候选择一个元素需要多个选择器搭配，这时候要算优先级就需要比对，比对规则如下：

  1. 先找优先级 100 的（id 选择器），id 选择器多的获胜，如果一样
  2. 找优先级 010 的，多的获胜，如果一样
  3. 以此类推

标记为重要（important）的声明。如下所示，在声明的后面加上 !important ，该声明就会被标记为重要的声明。

标记了 !important 的声明会被当作最高优先级的来源。<font color=red> 无论这个声明定义在哪，只要有 !important ，就是最高优先级，就按它渲染（除非和它冲突的也有 !important ）。</font>

# 注意点

一般不建议使用 !important ，因为如果要覆盖这个样式，又要用个 !important ，导致后面样式优先级越来越大，越来越难改

id 选择器同理，优先级也太大。

因此浏览器判断渲染哪个样式的流程一般如下：

  1. 首先看样式是默认样式还是用户定义的，优先选择用户定义的。如果是用户定义的→

  2. 是行内样式还是选择器内的，优先选择行内样式。如果是选择器→

  3. 判断选择器的优先级

    1. 首先看 id 选择器数量，选择数量多的，如果一样
    2. 看优先级 010（类选择器）的数量，选择数量多的，依次类推

    如果还是一样→

  4. 看源码顺序，哪个定义在后面优先选哪个。

  5. 额外规则：有 **!important 选！important**，如果冲突的样式也有 **!important**，则再按照以上规则择优

如果一个声明（针对某个样式的一行代码）在层叠中 “胜出”，它就被称作一个层叠值。元素的每个属性最多只有一个层叠值。

# 继承

子元素会继承父元素的部分样式，主要是文本相关的，如：

  • color：字体颜色
  • font：字体
    • font-family：字体（宋体、黑体...）
    • font-size：字体大小
    • font-weight：字体宽度（控制粗细）
    • font-variant：字母的大小写
    • font-style：字体风格（斜体..）
  • line-height：行高
  • text-align：对齐方式（水平）
  • text-indent：首行缩进
  • text-transform：也是字母大小写
  • white-space：是否超出换行（wrap）
  • letter-spacing：字体间距（类似行间距）
  • word-spacing：单词间距

还有一些别的特殊属性，如：

列表属性：list-style、list-style-type、 list-style-position 以及 list-style-image。

表格边框属性： border-collapse 和 border-spacing

# 继承属性冲突

继承属性会顺序传递给后代元素，直到它被层叠值覆盖。

也就是说，继承的优先级非常低，只要定义了新的样式，即可被覆盖

# 继承覆盖层叠 inherit

有时，我们想用继承代替一个层叠值。这时候可以用 inherit 关键字。可以用它来覆盖另 一个值，这样该元素就会继承其父元素的值

当然，前提是 inherit 的选择器优先级需要大于要覆盖的那个（相当于是生成新的层叠值，只不过引用了父元素的属性值）

这样做的好处是：相当于绑定了父元素，当父元素发生改变时，它也会跟着改变

# 重置样式 initial

有时，你需要撤销作用于某个元素的样式。这可以用 initial 关键字来实现。使用 initial 可以将其重置为默认值，这么做的好处是不需要思考太多。

<font color=red> 其实就是继承了 body 的样式 </font>

initial ：初始化

# 简写属性覆盖层叠值

简写属性是用于同时给多个属性赋值的属性。比如 font 是一个简写属性，可以用于设置多 种字体属性。它指定了 font-style 、 font-weight 、 font-size 、 font-height 以 及 font-family

简写属性会默默覆盖其他样式

简写属性可以省略一些值，只指定我们关注的值。但是要知道，这样做仍然会设置省略的值，即它们会被隐式地设置为初始值
