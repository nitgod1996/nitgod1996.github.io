相对单位和自适应

由于显示器的大小、分辨率的差别，同时 100px 在不同的用户页面上可能相差很大，每个用户的体验也可能千差万别。

不管我们 喜欢与否，都得抛弃以前那种固定宽度的栏目设计，开始考虑响应式设计

# em

em 是最常见的相对长度单位，适合基于特定的字号进行排版。在 CSS 中，1em 等于当前元素的字号，其准确值取决于作用的元素

可以看到， padding 是 16px 。这里设置内边距的值为 1em 。浏览器将其乘以字号，最终渲染为 16px 。这一点很重要：浏览器会根据相对单位的值计算出绝对值，称作计算值（computed value）。

由于 font-size 可以继承，我们不需要每次使用 em 时都去定义一下字号大小，它将沿用父元素的字号。

# 使用 em 定义字号

前面说了， 1em 等于元素当前的字号，而当没有显示地定义 font-size 时，字号大小继承自父元素。那么我们可以通过 em 定义元素的 font-size 为父元素的 n 倍。

如定义 font-size: 0.5em ，此时元素的字号即为父元素的二分之一

# rem

rem 是 root em 的缩写。 rem 和 em 差不多，也是跟随字号相对单位，但 rem 不是相对于当前元素，而是相对于根元素的单位。也就是 **<html>（可以通过 :root 选中）。一般浏览器默认根元素的字号是 16px**

与 em 相比， rem 降低了复杂性。实际上， rem 结合了 px 和 em 的优点，既保留了相对单位 的优势，又简单易用。但也不意味着抛弃 px 和 em

拿不准的时候，用 rem 设置字号，用 px 设置边框，用 em 设置其他大部分属性。

# 重置 html 默认字号

过去几年有一个常见的模式，更准确地说是反模式，就是将网页根元素的字号设置为 0.625em 或者 62.5%

这样会把默认字号缩小为 10px，可以极大的简化计算。

但也有缺点：可能会被迫写很多重复的代码，因为 10px 对大部分文字来说太小了，而 16px 则差不多刚刚好

应该摒弃像素思维，就把 rem 当做正常单位，不要总想着 1rem 等于多少 px。不要为了 10px 便于计算而设置默认字号为 10px，而是应该以实际情况设置最合适的默认字号如（14px、16px）。而不是将默认字体设置为 10px 然后再覆盖一遍

# 响应式尺寸

我们甚至可以根据屏幕尺寸，用媒体查询改变根元素的字号。这样就能够基 于不同用户的屏幕尺寸，渲染出不同大小的面板

媒体查询：即 @media 规则，可以指定某种屏幕尺寸或者媒体类型（比如，打印机 或者屏幕）下的样式。

（阿里开发的 flexible.js 就用到了这个，此外还同时监听了视口的变化。）

# 视口的相对单位

视口：浏览器窗口里网页可见部分的边框区域。它不包括浏览器的地址栏、工具 栏、状态栏。

视口有以下单位：

  • vh ：视口高度的 1/100。
  • vw ：视口宽度的 1/100。
  • vmin ：视口宽、高中较小的一方的 1/100
  • vmax ：视口宽、高中较大的一方的 1/100

可以通过视口单位设置浏览器窗口变大变小时组件大小跟着变化（自适应）的功能。

# 使用 vw 定义字号

通过相对视口单位设置字号，然后在通过 rem / em 设置组件的尺寸，那么就可以让组件随着视口大小改动。

也可以在一定程度上适配不同尺寸的设备，代替媒体查询的功能。同时可以在不同尺寸的设备间平滑过渡，而不是在某个断点突然改变。

# 百分比单位和无单位数值

除了 em、rem、视口单位，还有个相对单位就是百分比单位。

百分比就是相对父元素的百分比的大小，这个没什么好说的。

有些属性允许无单位的值（即一个不指定单位的数）。支持这种值的属性包括 line-height、 z-index、font-weight（700 等于 bold，400 等于 normal，等等）。

其中特别要注意的是行高 line-height 的设置

# 行高 line-height

我们常常使用 line-height 来设置字体的垂直居中，其实在设置行高时，通常我们应该使用无单位的数值

由于行高是属于会被继承的属性，如果使用了别的单位设置行高，那么每次修改元素字号时，都可能会发生行高不适配，字体重叠的情况。，如下
