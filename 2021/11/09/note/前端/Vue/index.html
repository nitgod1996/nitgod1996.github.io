# Vue 有什么用

# 1. MVVM 模式

model-view viewModel 模式，即模型 - 视图的双向绑定。

模型 model：可看作 JavaScript 代码

视图 view：可看作 HTML 的标签内容

在该模式下，view 随时监听 model，只要 model 发生了变化，view 也会跟着变化，Vue 就实现了这点。

  1. 在 HTML 标签中，以 { { message } } 表示文本，在 JavaScript 中绑定 message
  2. 当浏览器控制台输入 app.message=hello world 时，显示内容变为 hello world。这便是虚拟 DOM
  3. 用 jQuery 也可以改变显示内容，但会破坏视图 view，即原生 HTML 代码。Vue 就相当于把标签内容以变量的形式代替（ message ），然后在 new Vue(){ } 内对变量 message 进行赋值，只需修改变量的值即可

# 1.5 computed 和 methods 的区别

  • 方法（ methods ）则每次调用都会执行一遍函数

# 2. 生命周期函数

生命周期：每一个 vue 实例从创建到销毁的过程，就是这个 vue 实例的生命周期。在这个过程中，他经历了从开始创建、初始化数据、编译模板、挂载 Dom、渲染→更新→渲染、卸载等一系列过程。

这个写得比较详细：https://blog.csdn.net/weixin_42707287/article/details/111641286

  1. new Vue() 实例化一个 vue 实例，然后 init 初始化 event 和 lifecycle ，即事件和生命周期

  2. beforeCreate() 执行，可以在里面写这个时刻需要执行的代码

  3. init injections& reactivity ：初始化数据，定义 ** data 、 methods 、 computed 、 watch ** 这些属性方法。后续数据发生变化时，可以感知到数据的变化并完成页面渲染

  4. created() 执行，这时候可以在里面写有关 data 数据处理的代码，以及 methods 下的方法

  5. 进入条件判断：判断是否有 el （ el 是 Vue 实例挂载的元素节点，类似 css 选择器）

    1. 有→是否有 template
      1. 有→将 template 模板转换成 render 函数。（render 是渲染函数）
      2. 没有→直接将获取到的 el （也就是我们常见的 #app ）编译成 templae , 然后在将这个 template 转换成 render 函数。
    2. 没有→等待调用 $mount(el) 方法

  6. beforeMount() 执行

  7. beforeMount 调用后，生产一个虚拟 dom（ vm.$el ）用 render 渲染后替换掉 $el 。

    5、6、7 主要就是渲染 html 页面

  8. mounted() 执行，可以写操作 dom 的代码在这个函数内，因为这个时候 dom 已经渲染完成了。

  9. 状态数据 data 发生变化时，触发 beforeUpdate() ，准备重新渲染新数据

  10. 生成新的虚拟 DOM 和更新 render 函数，将虚拟 DOM 渲染成真实 DOM

  11. update() 执行，这里可以操作新的 DOM

  12. 之后实例销毁前， beforeDestroy() 执行，此时仍可以操作实例

  13. 一系列的销毁动作

  14. 销毁完成，执行 destroy()

生命周期函数的 this 上下文指向调用它的 Vue 实例。

# 生命周期函数定义不要使用箭头函数

经常导致 Uncaught TypeError: Cannot read property of undefined 或 Uncaught TypeError: this.myMethod is not a function 之类的错误。

  • 标准函数： this 返回的是调用该函数的对象的域
  • 箭头函数： this 返回的是定义该函数的域

总之，箭头函数在定义好后 this 返回就已经定死了，赋值给别的函数也不行

生命周期函数是在 vue 实例的生命周期中调用的，应该用标准函数声明

  • { { message } } 是纯文本，不会渲染 html 代码

    但可以通过 v-html 设置成 html 代码并渲染（虽然不推荐，可能导致 XSS 攻击）

# 2.6 v-bind 绑定属性

v-bind 是绑定标签属性的指令，如 <img v-bind:src="imageSrc"> ， imageSrc 便是 src 的值， imageSrc 可以在 Vue 实例中定义。

语法：

  • v-bind:属性名="变量"

    • 提几句，这个属性名仍是对应 html 标签的属性名，如 src 本就是 <img> 上有的

# 4. 指令对比

# 4.1 v-if vs v-show

v-if 是 “真正” 的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建。

v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做 —— 直到条件第一次变为真时，才会开始渲染条件块。

相比之下， v-show 就简单得多 —— 不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。

<font color=red> 一般来说， v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。</font>

# 4.2 v-if 与 v-for 一起使用

当 v-if 与 v-for 一起使用时， v-for 具有比 v-if 更高的优先级。请查阅列表渲染指南以获取详细信息。

# vue-cli

Vue CLI 是一个基于 Vue.js 进行快速开发的完整系统，提供：

  • 通过 @vue/cli 实现的交互式的项目脚手架。（其实就是类似后端 spring 的框架，一些必要的东西（架子）都给我们写好了）

  • 通过 @vue/cli + @vue/cli-service-global 实现的零配置原型开发。

  • 一个运行时依赖 (

    @vue/cli-service

    )，该依赖：

    • 可升级；
    • 基于 webpack 构建，并带有合理的默认配置；
    • 可以通过项目内的配置文件进行配置；
    • 可以通过插件进行扩展。

  • 一个丰富的官方插件集合，集成了前端生态中最好的工具。

  • 一套完全图形化的创建和管理 Vue.js 项目的用户界面。

# 3. 项目目录解析

项目文件创建好后，是这样一个目录

  • node_modules： npm 的包管理仓库，存放所有由 npm down 下来的插件

  • public：不知道

  • src★：项目源目录，所有项目内容相关的文件包括 .js 、 .css 、 .vue 等文件都放在这里

    • assets：一般用来存放样式（css、less、sass）、js 等文件
    • components：组件目录。存放项目组件文件 .vue
    • router：路由相关文件。组件跳转。（一开始没有，要用的时候自己创建）
    • store：vuex 相关文件。组件状态通信。（一开始没有，要用的时候自己创建）
    • ...：还有一些其他的比如 echarts 等需要用到都建议放这个目录
    • App.vue ：项目根组件
    • main.js ：项目入口

  • static：存放静态资源。一般用来存放图片、json 数据等。（一开始没有，要用的时候自己创建）

  • config：项目配置文件。（一开始没有，要用的时候自己创建）

    • index.js ：项目一些配置变量
    • dev.env.js ：开发环境变量
    • prod.env.js ：生产环境变量
    • test.env.js ：测试环境变量

  • .gitignore ：git 上传需要忽略的文件配置

  • babel.config.js ：不知道

  • jsconfig.json ：不知道

  • package-lock.json ：不知道

  • package.json ：项目基本信息。（所需模块、项目名称、版本等都在里面，类似 maven 的 pom.xml ）

  • README.md ：项目说明

  • vue.config.js ：项目其他配置信息。如 webpack 配置
