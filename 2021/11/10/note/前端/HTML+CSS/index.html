# HTML

# 1. 简介

  • <!DOCTYPE html> 声明为 HTML5 文档
  • <html> 元素是 HTML 页面的根元素
  • <head> 元素包含了文档的元（meta）数据，如 <meta charset="utf-8"> 定义网页编码格式为 utf-8。
  • <title> 元素描述了文档的标题
  • <body> 元素包含了可见的页面内容
  • <h1> 元素定义一个大标题
  • <p> 元素定义一个段落

# 2. 标签

# 2.1 标签类型

标签，也称元素，分为内联元素和块级元素。

块级元素会换行，而内联元素不会，可以通过 CSS 的 display 属性修改设置。

# 2.1.1 内联元素

​ 内联元素又名行内元素，和其对应的是块元素 (block element)，都是 html 规范中的概念。内联元素的显示，为了帮助理解，可以形象的称为 “文本模式”，即一个挨着一个，都在同一行按从左至右的顺序显示，不单独占一行。

任何不是块级元素的可见元素都是内联元素。其表现的特性是 “行布局” 形式，这里的 “行布局” 的意思就是说其表现形式始终以行进行显示。

内联元素有以下特点：

  1. 和其他元素都在一行上，遇到父级元素边界会自动换行
  2. 高、行高以及内外边距都不可以改变
  3. 宽度与内容一样宽，且不可改变
  4. 行内元素只能容纳文本或者其他行内元素
  5. 对于行内元素，需要注意的是：设置宽度 width 无效，设置高度无效，可以通过设置 line-height 来设置，设置 margin 只有左右有效，上下无效，设置 padding 只有左右有效，上下无效

常见的内联元素有：

  • a – 锚点
  • abbr – 缩写
  • b – 粗体 (不推荐)
  • big – 大字体
  • br – 换行
  • cite – 引用
  • font – 字体设定 (不推荐)
  • i – 斜体
  • img – 图片
  • input – 输入框
  • label – 表格标签
  • select – 项目选择
  • small – 小字体文本
  • sub – 下标
  • sup – 上标
  • textarea – 多行文本输入框
  • u – 下划线

# 2.1.2 块元素

块元素和内联元素的基本差异是块元素一般都从新行开始，相邻的块级元素将会在不同行显示。

块级元素的特点：

  1. 独霸一行，总是在新行上开始
  2. 宽度缺省是它父级元素的 100%，除非设定一个宽度
  3. 高度、行高、外边距、内边距都可以设置
  4. 可以容纳其他内联元素或者其他块元素

以下是常见的块级元素：

  • address – 地址
  • center – 举中对齐块
  • dir – 目录列表
  • div – 常用块级容易，也是 CSS layout 的主要标签
  • dl – 定义列表
  • form – 交互表单
  • h1~h6 – 标题
  • hr – 水平分隔线
  • menu – 菜单列表
  • ol – 有序表单
  • p – 段落
  • table – 表格
  • ul – 无序列表

# 2.7 <div>/<span> 无样式标签

<div> 和 <span> 没有特殊定义。

<div> 和 <span> 是最常用到的两个标签，因为 html 大部分时候仅仅写个文本就够了，排版、样式都是由 CSS 完成的。

<div> 和 <span> 的唯一区别是

  • <div> 是块级元素，单独成一行（也就是会自动换行）。适合做盒子或者对网页布局。
  • <span> 是内联元素，不会自动换行。可以做文本容器.

一般用法是以 <div> 包裹 <span>

# 2.5.1 优先级判断

权重记忆口诀：

从 0 开始：

一个行内样式 + 1000；

一个 id 选择器 + 100；

一个属性、class 或者伪类：：选择器 + 10；

一个元素、或者：伪元素选择器 + 1；

* 通配符 + 0。

行内样式是直接写在 HTML 标签上的样式

  • !important 声明的样式的优先级最高；

    • !important 是提升样式优先级的指令，不建议经常使用
    • 2 个！important 冲突时，权重大的优先级高

  • 如果优先级相同，则最后出现的样式生效；

  • 继承得到的样式的优先级最低；

  • 通用选择器（*）、子选择器（>）和相邻同胞选择器（+）并不在这四个等级中，所以它们的权值 都为 0 ；

  • 样式表的来源不同时，优先级顺序为：内联样式 and 外联样式 > 内部样式 > 外部样式 > 浏览器用户自定义样式 > 浏览器默认样式。

# 3.1.7.1 内联元素

​ 内联元素又名行内元素，和其对应的是块元素 (block element)，都是 html 规范中的概念。内联元素的显示，为了帮助理解，可以形象的称为 “文本模式”，即一个挨着一个，都在同一行按从左至右的顺序显示，不单独占一行。

任何不是块级元素的可见元素都是内联元素。其表现的特性是 “行布局” 形式，这里的 “行布局” 的意思就是说其表现形式始终以行进行显示。

内联元素有以下特点：

  1. 和其他元素都在一行上，遇到父级元素边界会自动换行
  2. 高、行高以及内外边距都不可以改变
  3. 宽度与内容一样宽，且不可改变
  4. 行内元素只能容纳文本或者其他行内元素
  5. 对于行内元素，需要注意的是：设置宽度 width 无效，设置高度无效，可以通过设置 line-height 来设置，设置 margin 只有左右有效，上下无效，设置 padding 只有左右有效，上下无效

# CSS 框架

# 1. css 基础框盒模型（盒子模型）

CSS 传统布局基于盒子模型，所有元素都在一个盒子（容器）中

每个盒子有 4 个边界：

  1. 内容边界 Content edge（最里面显示内容）
  2. 内边距边界 Padding Edge（content + 内边距 padding）
  3. 边框边界 Border Edge（padding edge + 边框宽度 border width）
  4. 外边框边界 Margin Edge（border edge + 外边距 margin）

# 1.1 内容边界

容纳着元素的 “真实内容”，例如文本、图像等，其尺寸为内容宽度和内容高度确定。

主要 css 属性：

  • width ：内容宽度。% 或 px 或 rem。% 是相对父元素的百分比

  • height ：内容高度。% 或 px 或 rem

  • min-width ：最小内容宽度。这种一般是针对可以灵活变动的元素

  • max-width ：最大内容宽度

  • min-height ：最小内容高度

  • max-height ：最大内容高度

  • box-sizing ：

    • content-box ：默认。在宽度和高度之外绘制元素的内边距和边框。<font color=red>height 和 width 绑定内容边界 </font>
    • border-box ：向内绘制内边距和边框。换句话说，<font color=red>height 和 width 绑定边框边界 </font>

# 1.2 内边距边界

内容边界 + padding。是边框（border）内的那块空间。内边距则是内容和边框之间的那块空白区域

主要 css 属性：

  • padding-top ：上内边距。px、%、rem
  • padding-right ：右内边距。
  • padding-bottom ：下内边距
  • padding-left ：左内边距
  • padding ：内边距，以上的简写

# 1.3 边框边界

内边距边界 + 边框宽度。边框由 border 属性控制

主要 css 属性：

  • border-width ：边框宽度。
  • border-style ：边框样式：
    • dotted 定义点状
    • dashed 定义虚线
    • solid 定义实线。
    • double 定义双线
  • border-color ：边框颜色
  • border ：以上简写
  • 可以分别控制各边边框
    • border-top ：上边框
      • border-top-width ：上边框宽度
  • box-sizing ：如果 box-sizing 设置 border-box ，则可用 width 和 height 设置边框边界

# 1.4 外边距边界

盒子最外围边界，边框边界 + margin。外边距是边框与别的盒子之间的空白区域

主要 css 属性：

  • margin-top ：上外边距。px、%、rem
  • margin-right ：右外边距。
  • margin-bottom ：下外边距
  • margin-left ：左外边距
  • margin ：外边距，以上的简写

# 1.4.1 外边距重叠

相邻的盒子的外边距会重叠，取较大的边距。（其实就是把外边距视为自己边框 - 相邻盒子边框的距离）。

如果不想折叠，则设定 float 和 position=absolute 的元素不会产生外边距重叠行为。

<font color=red> 只有上下外边距会重叠，左右不会 </font>

# 3.1.7.1 内联元素

​ 内联元素又名行内元素，和其对应的是块元素 (block element)，都是 html 规范中的概念。内联元素的显示，为了帮助理解，可以形象的称为 “文本模式”，即一个挨着一个，都在同一行按从左至右的顺序显示，不单独占一行。

任何不是块级元素的可见元素都是内联元素。其表现的特性是 “行布局” 形式，这里的 “行布局” 的意思就是说其表现形式始终以行进行显示。

内联元素有以下特点：

  1. 和其他元素都在一行上，遇到父级元素边界会自动换行
  2. 高、行高以及内外边距都不可以改变
  3. 宽度与内容一样宽，且不可改变
  4. 行内元素只能容纳文本或者其他行内元素
  5. 对于行内元素，需要注意的是：设置宽度 width 无效，设置高度无效，可以通过设置 line-height 来设置，设置 margin 只有左右有效，上下无效，设置 padding 只有左右有效，上下无效

块元素和内联元素的基本差异是块元素一般都从新行开始，相邻的块级元素将会在不同行显示。

块级元素的特点：

  1. 独霸一行，总是在新行上开始
  2. 宽度缺省是它父级元素的 100%，除非设定一个宽度
  3. 高度、行高、外边距、内边距都可以设置
  4. 可以容纳其他内联元素或者其他块元素

如果你创造了一个 <div> 没有设置 box-sizing 属性为 border-box (不设置的话默认值为 content-box.)，同时你设置 width:100px; border:10px solid red; padding:10px; 那么最终 div 容器的实际宽度为：

100px(width)+2*10px*(padding)+2*10px(border)=140px

所以你会得到一个比你预期（100px）还要更大的容器，结果就是会破坏网页布局。

# 定位 position

​ 指定了元素的定位类型。通常与 left 等定位属性配合使用

  • position

    • relative：相对于正常位置移动， left 等属性设置的是移动值。

    • fixed：相对浏览器窗口固定，即使窗口滚动也不会动。适用于做浮窗

    • absolute：绝对定位，其实是相对于父元素的坐标， left 等属性设置的是坐标值。

      1. 设置了 absolute 的元素如果存在有祖先元素设置了 position 属性为 relative 或者 absolute，则这时元素的定位对象为此已设置 position 属性的祖先元素。
      2. 如果并没有设置了 position 属性的祖先元素，则此时相对于 body 进行定位。
