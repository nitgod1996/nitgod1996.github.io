# 一、 REST

# 1. 什么是 REST 以及 RESTful

REST 指的是一组架构约束条件和原则。" 如果一个架构符合 REST 的约束条件和原则，我们就称它为 RESTful 架构。

​ 相当于是编写 API 的一套规则，像高德地图提供的 API。

URL https://restapi.amap.com/v3/place/text?parameters
请求方式 GET

上面是高德地图提供的 API，返回的是 json 格式的数据信息。

以下介绍 REST 的几大概念：

  • 资源与 URI
  • 统一资源接口
  • 资源的表述
  • 资源的链接
  • 状态的转移

# 1.1 资源与 URI

URI 是用于标记网络上的资源的唯一标识

要让一个资源可以被识别，需要有个唯一标识，在 Web 中这个唯一标识就是 URI (Uniform Resource Identifier)。

URI 既可以看成是资源的地址，也可以看成是资源的名称。

以下是 URI 的编写技巧：

  • 使用_或 - 分隔单词。如 http://www.oschina.net/news/38119/oschina-translate-reward-plan。
  • 使用 / 来表示资源的层级关系
  • 使用？用来参数传递，& 分隔参数
    • 如 https://restapi.amap.com/v3/place/text?keywords = 北京大学 & city=beijing&output=xml&offset=20&page=1&key=<用户的 key>&extensions=all
  • , 或；可以用来表示同级资源的关系

# 1.2 统一资源接口

RESTful 架构应该遵循统一接口原则，接口应该使用标准的 HTTP 方法如 GET，PUT 和 POST，并遵循这些方法的语义。

# 1.3 资源的表述

客户端通过 HTTP 方法可以获取资源。确切的说，客户端获取的只是资源的表述而已。

​ 资源在外界的具体呈现，可以有多种形式。在客户端和服务端之间传送的也是资源的表述，而不是资源本身。

​ 例如文本资源可以采用 html、xml、json 等格式，图片可以使用 PNG 或 JPG 展现出来。

资源的表述包括数据和元数据（描述数据的数据），例如，HTTP 头 "Content-Type" 就是这样一个元数据属性。

​ 客户端需要在 Request 头通过 Accept 说明自己需要的格式，而服务端在 Content-Type 返回该格式。如果不支持返回该格式，则报 406 错误

# 二、 RPC

# 1. 什么是 RPC

RPC：Remote Procedure Call 远程过程调用

RPC 就是要像调用本地的函数一样去调远程函数。

在远程调用时，我们需要执行的函数体是在远程的机器上的。以下是一些 RPC 的概念

# 1.1 Call ID 映射

​ 我们如何告诉远程机器我们要调用哪个函数？

  • 在本地调用中：函数体是直接通过函数指针来指定的。

  • 在 RPC 中：每个函数都有自己的一个 ID，这个 ID 是所有进程中都是唯一确定的。客户端在做远程过程调用时，必须附上这个 ID。

# 1.2 序列化和反序列化

​ 客户端怎么把参数值传给远程的函数呢？

  • 本地调用中：我们只需要把参数压到栈里，然后让函数自己去栈里读就行。
  • RPC 中：需要客户端把参数先转成一个字节流，传给服务端后，再把字节流转成自己能读取的格式。这个过程叫序列化和反序列化。

# 1.3 网络传输

​ 客户端和服务端是通过网络连接的。所有的数据都需要通过网络传输，因此就需要有一个网络传输层。

​ 尽管大部分 RPC 框架都使用 TCP 协议，但其实 UDP 也可以。

# 2. RPC 与 REST 的区别

REST 是一种架构风格，指的是一组架构约束条件和原则。满足这些约束条件和原则的应用程序或设计就是 RESTful。

大部分的 REST 的实现中使用了 RPC 的机制，大致与有三部分组成：

  1. method：访问方式， get ， post 之类的

  2. Host/post：URI (统一资源标识)、服务器，端口

  3. Path：路径

RPC 是远程过程调用，就是像调用本地方法一样调用远程方法

通常的调用过程，把函数序列化，远端收到后，再把函数反序列化，完成函数调用。

比较项 REST RPC
通信协议 HTTP 一般使用 TCP
性能 低 高
灵活度 高 低

# 三、 直接使用 Runtime (推荐)

​ 在 Java 中如果需要调用第三方程序 / 脚本，可以直接通过 Runtime 实现，这也是最直接最粗暴的做法。

# 1.2 java 代码

package com.example.simpleDemo;
import java.io.BufferedReader;
import java.io.File;
import java.io.IOException;
import java.io.InputStreamReader;

 
public class RuntimeDemo02 {

    public static void main(String[] args) {
        String executer = "python";
        // python绝对路径
        String file_path = "F:\\project\\demo01\\Java_connect_Python\\runtimeDemo2.py";
        String[] command_line = new String[] {executer, file_path};
        try {
          //  Process process = Runtime.getRuntime().exec(command_line);
            Process process = Runtime.getRuntime().exec(command_line,null,new File("F:\\project\\demo01\\Java_connect_Python"));
            BufferedReader in = new BufferedReader(new InputStreamReader(process.getInputStream(),"GBK"));
            String line;
            while ((line = in.readLine()) != null) {
                System.out.println(line);
            }
            in.close();
            // java代码中的 process.waitFor() 返回值（和我们通常意义上见到的0与1定义正好相反）
            // 返回值为0 - 表示调用python脚本成功；
            // 返回值为1 - 表示调用python脚本失败。
            int re = process.waitFor();
            System.out.println("调用 python 脚本是否成功：" + re);
        } catch (IOException e1) {
            e1.printStackTrace();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

package com.example.simpleDemo;

import java.io.DataInputStream;
import java.io.IOException;
import java.io.InputStream;

public class RuntimeDemo01 {
    public static void main(String[] args) throws IOException, InterruptedException {
        String exe = "python";
        String command = "F:\\project\\demo01\\Java_connect_Python\\add.py";
        String num1 = "1";
        String num2 = "2";
        String[] cmdArr = new String[] {exe, command, num1, num2};
        Process process = Runtime.getRuntime().exec(cmdArr);
        InputStream is = process.getInputStream();
        DataInputStream dis = new DataInputStream(is);
        String str = dis.readLine();
        process.waitFor();
        System.out.println(str);
    }
}

  1. 在 py 文件添加

    import sys

# 3.3 函数解析

# 3.3.1 类 Runtime

每个 Java 应用程序都有一个 Runtime 类实例，使应用程序能够与其运行的环境相连接。可以通过 getRuntime 方法获取当前运行时。

返回值类型 函数
static Runtime Runtime.getRuntime() <br/> 返回与当前 Java 应用程序相关的运行时对象。
Process Runtime.getRuntime().exec(String[] cmdarray, String[] envp, File dir) <br> 在指定环境和工作目录的独立进程中执行指定的命令和变量。

# 3.3.2 类 Process

可通过 ProcessBuilder.start() 和 Runtime.exec 方法创建一个本机进程，并返回 Process 子类的一个实例，该实例可用来控制进程并获得相关信息。

Process 类提供了执行从进程输入、执行输出到进程、等待进程完成、检查进程的退出状态以及销毁（杀掉）进程的方法。

返回值类型 函数（以下 process 是 Process 的对象）
abstract InputStream process.getInputStream() <br/> 获取子进程的输入流。
abstract OutputStream process.getOutputStream() <br> 获取子进程的输出流。
abstract int <br> 返回：进程的出口值。<br>0 表示正常终止，1 表示有错误 process.waitFor() <br> 导致当前线程等待，如有必要，一直要等到由该 Process 对象表示的进程已经终止。

# 3.4 总结

比起前面的，显然 Java 自带的 Runtime 更加方便、简单粗暴。此方法适用于离线项目。

  1. 优点：
    1. 简单、方便
    2. 不需要部署，相当于直接内嵌在 Java 项目
    3. jdk 自带功能，不需要其他外部依赖支持
  2. 缺点：
    1. 耦合度过大，每次调用 Python 脚本都需要重写一次
