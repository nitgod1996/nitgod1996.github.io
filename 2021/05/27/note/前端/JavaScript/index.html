# 2. 引入 JavaScript

<script> 标签：通过此标签引入 js 代码

# 2.1 内嵌入 html

直接在 <script></script> 中间写代码。

<script> 引入位置会影响执行顺序，一般放在 <body> 的最后，这样不会影响到页面的渲染，不然可能会等 js 加载，此时页面是空白的

同一页面的多个 <script> 会合并为一个域

# 2.2 外部导入 js 文件

单独建一个 .js 文件，在 html 中以 <script src="**.js"></script> 引入。

属性：

  • src ：js 文件的位置，可以是本地（自己写的或下载的），也可以是网上的源
  • defer ：延迟执行脚本。等页面渲染完成后解析脚本
  • async ：异步。加载 js 和渲染页面各干各的，互不干扰。（不推荐，有些浏览器不支持）

前面说了， <script> 引入位置会影响执行顺序，一般放在 <body> 的最后。但如果加了 defer 或 async ，即使放前面也不影响页面渲染。

0.6 * 2 = 1.2 # 1
0.2 * 2 = 0.4 # 0
.....

    从上面可以看出，0.1 的二进制格式是：0.0001100011....。这是一个二进制无限循环小数

# 3.3.3 字符数量匹配

量词 描述
n+ 匹配任何包含至少一个 n 的字符串。 <font color=green> 例如，/a+/ 匹配 "candy" 中的 "a"，"caaaaaaandy" 中所有的 "a"。 </font>
n* 匹配任何包含零个或多个 n 的字符串。 <font color=green> 例如，/bo*/ 匹配 "A ghost booooed" 中的 "boooo"，"A bird warbled" 中的 "b"，但是不匹配 "A goat grunted"。 </font>
n? 匹配任何包含零个或一个 n 的字符串。 <font color=green> 例如，/e?le?/ 匹配 "angel" 中的 "el"，"angle" 中的 "le"。 </font>
n 匹配包含 X 个 n 的序列的字符串。 <font color=green> 例如，/a {2}/ 不匹配 "candy," 中的 "a"，但是匹配 "caandy," 中的两个 "a"，且匹配 "caaandy." 中的前两个 "a"。 </font>
n X 是一个正整数。前面的模式 n 连续出现至少 X 次时匹配。 <font color=green> 例如，/a {2,}/ 不匹配 "candy" 中的 "a"，但是匹配 "caandy" 和 "caaaaaaandy." 中所有的 "a"。 </font>
n X 和 Y 为正整数。前面的模式 n 连续出现至少 X 次，至多 Y 次时匹配。 <font color=green> 例如，/a {1,3}/ 不匹配 "cndy"，匹配 "candy," 中的 "a"，"caandy," 中的两个 "a"，匹配 "caaaaaaandy" 中的前面三个 "a"。注意，当匹配 "caaaaaaandy" 时，即使原始字符串拥有更多的 "a"，匹配项也是 "aaa"。 </font>
n$ 匹配任何结尾为 n 的字符串。
^n 匹配任何开头为 n 的字符串。
?=n 匹配任何其后紧接指定字符串 n 的字符串。
?!n 匹配任何其后没有紧接指定字符串 n 的字符串。
