时过境迁，距第一次使用 maven 已经将近一年了，最近发现 maven 还有许多地方没搞明白，所以再继续学习一下，也当是复习巩固了。

# 1. maven 的作用

  • 依赖管理：
    • 在 pom.xml 中对项目需要的各种依赖（jar 包等）统一管理。
    • 如，当项目比较大时，依赖管理会变得非常臃肿。通过 maven 进行依赖管理
  • 项目构建：
    • 对项目进行编译、测试、打包、部署、上传到私服等。
    • 如，项目比较大时，需要分多个 module。如订单模块、支付模块、商品模块...。一般情况多个模块无法独立运行，需要借助 maven 的一键打包
  • Gradle 也是和 maven 一样的项目管理工具。

# 2. maven 安装

  1. maven 是 Java 项目，必须先安装 jdk。

  2. 下载 maven。官网 https://maven.apache.org/download.cgi。也可以去 csdn 下载，https://mp.csdn.net/console/upDetailed。

  3. 解压并配置环境变量。

    在 path 中添加上 D:\Java\apache-maven-3.6.2\bin。也可以设置 MAVEN_HOME。

  4. 检验。cmd 打开控制台，输入 mvn -v

# 3. maven 配置

​ 在 IDEA 中有默认的 maven 配置，本地仓库默认是在当前用户名.m2\repository。自定义配置可以与 IDEA 解耦，如果同时使用 eclipse 则将 maven 分离出来管理比较方便。

​ maven 仓库分为本地仓库、私服仓库和中央仓库，如下

仓库类型 说明
本地仓库 在自己电脑上存放用于项目的 jar 包。本地项目优先去本地仓库寻找依赖 jar 包
私服仓库 公司内部搭建的 maven 私服。一般存放公司项目所需 jar 和自己开发的 jar。
中央仓库 由 Apache 团队维护，包含了大部分的 jar，本地仓库没有的依赖一般有限都是去中央仓库下载。我们一般用国内镜像替代中央仓库。

# 3.1 - 配置本地仓库

  1. 在指定位置新建一个本地仓库文件夹，如 F:\serverCK\MavenRepository

  2. 在 conf\setting.xml 配置该目录

# 3.2 - 镜像配置

​ 默认的中央仓库下载依赖比较慢，一般配置阿里的镜像。也是在 conf\setting.xml 配置

# 4. 使用 IDEA 构建 maven 项目

​ 使用 IDEA 创建 maven 项目的方法有很多，既可以直接创建一个空的 maven project，也可以创建 springboot 项目时勾选 maven，就不一一赘述了。

​ maven 项目创建好后，最主要的就是主目录下的 pom.xml 和右侧的 maven 导航。

  • pom.xml 是 maven 的主要配置文件，其中定义了该项目的所有配置，主要包括

    • <dependencies> ：项目所需的外部依赖。 dependency 也称坐标

    • <packaging>jar</packaging> ：打包方式。springboot 打成 jar 包即可。普通的 web 项目需要打包 war 包。

    • <groupId>、<artifactId>、<version> ：项目信息，版本等

      • groupId：组织 ID
      • artifactId：项目在组织中的 ID
      • version：项目版本
      • scope：依赖范围。 test 表示仅测试有效。 runtime 是编译时不生效，运行时生效，如 jdbc 驱动。 complie 全部有效

    • <parent> ：项目继承的父依赖，如

      该项目继承了 org.springframework.boot 的所有依赖。

    • <build><plugins> ：maven 构建项目所需插件

    • <properties> ：编码及各依赖版本信息

  • install 项目打包：双击右侧的 install，将该项目打成 jar/war 包放在本地仓库，其他项目即可导入该项目的 jar（通过 groupId 等信息）。install 包含了 package 功能。
